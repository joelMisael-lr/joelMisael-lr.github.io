1. Modelos de arquitecturas de cómputo

1.1 Arquitecturas Clásicas

Las arquitecturas clásicas se basan en el modelo de Von Neumann, que consta de:

  • Una unidad central de procesamiento (CPU)
  • Memoria principal
  • Dispositivos de entrada/salida
  • Buses de comunicación

Característica clave: En la arquitectura Von Neumann, las instrucciones y los datos se almacenan en la misma memoria, lo que puede crear el "cuello de botella de Von Neumann" cuando la CPU debe esperar para acceder a la memoria.

1.2 Arquitecturas Segmentadas

Las arquitecturas segmentadas dividen el procesamiento de instrucciones en etapas que se ejecutan en paralelo:

Pipeline de 5 etapas

  1. Fetch (búsqueda)
  2. Decode (decodificación)
  3. Execute (ejecución)
  4. Memory (acceso a memoria)
  5. Writeback (escritura)

Ventajas

  • Mayor rendimiento
  • Uso eficiente de recursos
  • Mayor throughput

Desventajas

  • Complejidad de diseño
  • Riesgo de dependencias
  • Mayor consumo energético

1.3 Arquitecturas de Multiprocesamiento

Estas arquitecturas utilizan múltiples procesadores para ejecutar tareas en paralelo:

Tipo Descripción Ejemplo
SMP (Simétrico) Todos los procesadores comparten memoria y son iguales Servidores empresariales
NUMA Acceso no uniforme a memoria Sistemas de alto rendimiento
Cluster Múltiples computadoras trabajando juntas Supercomputadoras
GPU Miles de núcleos para procesamiento paralelo Procesamiento gráfico y científico

2. Análisis de los componentes

2.1 Arquitecturas

2.1.1 Unidad Central de Procesamiento (CPU)

La CPU es el cerebro de la computadora, compuesta por:

  • Unidad de control: Coordina las operaciones
  • ALU: Realiza operaciones aritméticas y lógicas
  • Registros: Almacenamiento temporal rápido
  • Cache: Memoria rápida para datos frecuentes

2.1.2 Unidad Aritmética Lógica (ALU)

Componente que realiza operaciones matemáticas y lógicas:

Operaciones Aritméticas

  • Suma
  • Resta
  • Multiplicación
  • División

Operaciones Lógicas

  • AND
  • OR
  • NOT
  • XOR

Características

  • Operaciones con enteros
  • Banderas de estado
  • Diseño en cascada

2.1.3 Registros

Pequeñas unidades de almacenamiento ultrarrápido dentro de la CPU:

Registro Función Tamaño típico
AC (Acumulador) Almacena resultados de operaciones 32/64 bits
PC (Contador de Programa) Dirección de la siguiente instrucción 32/64 bits
IR (Registro de Instrucción) Almacena la instrucción actual 32/64 bits
MAR (Registro de Dirección de Memoria) Dirección a leer/escribir en memoria 32/64 bits

2.1.4 Buses

Canales de comunicación entre componentes:

Bus de Datos

Transporta datos entre CPU, memoria y periféricos. Bidireccional.

Bus de Direcciones

Especifica ubicaciones de memoria. Unidireccional (CPU → otros).

Bus de Control

Transmite señales de control y temporización.

2.2 Memoria

2.2.1 Conceptos básicos del manejo de memoria

Principios fundamentales:

  • Jerarquía de memoria: Registros → Cache → RAM → Almacenamiento
  • Localidad: Temporal y espacial
  • Direccionamiento: Físico vs. lógico
  • Protección: Aislamiento de procesos

2.2.2 Memoria principal

Características de la memoria RAM:

DRAM

  • Dinámica (requiere refresco)
  • Mayor densidad
  • Más económica
  • Usada en memoria principal

SRAM

  • Estática (no requiere refresco)
  • Más rápida
  • Menor densidad
  • Usada en caché

Nuevas tecnologías

  • DDR4/DDR5
  • Memoria no volátil
  • 3D XPoint
  • HBM (High Bandwidth)

2.2.3 Memoria caché

Memoria pequeña y rápida que almacena datos frecuentes:

Nivel Tamaño Velocidad Ubicación
L1 32-64 KB Más rápida Dentro del núcleo
L2 256 KB - 1 MB Rápida Núcleo o compartida
L3 2-32 MB Menos rápida Compartida entre núcleos

2.3 Manejo de la entrada/salida

2.3.1 Módulos de entrada/salida

Componentes que gestionan la comunicación con periféricos:

  • Controlador: Gestiona el dispositivo específico
  • Buffer: Almacena datos temporalmente
  • Lógica de control: Coordina la transferencia
  • Registros: Almacenan estado y comandos

2.3.2 Entrada/salida programada

La CPU controla directamente la transferencia:

  1. CPU verifica estado del dispositivo
  2. Cuando está listo, CPU transfiere datos
  3. Repite para cada byte/palabra
  4. Ineficiente para grandes transferencias

2.3.3 Entrada/salida mediante interrupciones

El dispositivo notifica a la CPU cuando está listo:

1. Solicitud

CPU inicia operación E/S y continúa ejecutando

2. Interrupción

Dispositivo envía señal de interrupción cuando está listo

3. Procesamiento

CPU salva estado, atiende interrupción y luego retorna

2.3.4 Acceso directo a memoria (DMA)

Controlador especial que maneja transferencias sin CPU:

Ventajas

  • Libera a la CPU para otras tareas
  • Mayor velocidad de transferencia
  • Eficiente para grandes bloques

Proceso

  1. CPU configura DMA
  2. DMA gestiona transferencia
  3. DMA interrumpe al finalizar

Aplicaciones

  • Transferencia de disco
  • Gráficos
  • Redes
  • Audio

2.3.5 Canales y procesadores de E/S

Sistemas avanzados para gestión de E/S:

Tipo Descripción Ejemplo
Canal selector Gestiona un dispositivo a la vez a alta velocidad Discos de alta gama
Canal multiplexor Gestiona múltiples dispositivos de baja velocidad Terminales, impresoras
Procesador de E/S CPU dedicada para E/S con su propio conjunto de instrucciones Sistemas mainframe

2.4 Buses

2.4.1 Tipos de buses

Bus del sistema

  • Conecta CPU con memoria
  • Alta velocidad
  • Ejemplo: FSB (Front Side Bus)

Bus de E/S

  • Conecta periféricos
  • Menor velocidad
  • Ejemplo: PCI, USB

Bus de expansión

  • Para tarjetas adicionales
  • Ejemplo: PCI Express

2.4.2 Estructura de los buses

Componentes principales:

  • Líneas de datos: Transportan información
  • Líneas de direcciones: Especifican ubicación
  • Líneas de control: Señales de temporización y estado
  • Arbitraje: Mecanismo para resolver conflictos

2.4.3 Jerarquía de los buses

Organización moderna de buses:

Bus de CPU

Máxima velocidad (GHz), conecta núcleos, cache y controlador de memoria

Bus de memoria

Alta velocidad, conecta CPU con RAM (DDR4/DDR5)

Bus de expansión

PCI Express para tarjetas gráficas y dispositivos rápidos

Bus de periféricos

USB, SATA para dispositivos de almacenamiento y E/S

2.4.4 Interrupciones

Mecanismo para notificar eventos importantes:

Tipo Origen Ejemplo
Hardware Dispositivos físicos Teclado, temporizador
Software Instrucciones especiales Llamadas al sistema
Excepciones Errores de ejecución División por cero

Proceso de atención de interrupciones: Salvar contexto → Identificar interrupción → Ejecutar rutina → Restaurar contexto → Retornar
